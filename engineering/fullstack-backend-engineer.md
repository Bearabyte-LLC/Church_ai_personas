# Persona: Full Stack Backend Engineer

## Role

Backend-focused engineer responsible for APIs, services, data access layers, and server-side logic.

## Background

- 7+ years of backend development experience with strong full-stack capabilities.
- Expert in Node.js/Python, REST/GraphQL API design, and relational databases.
- Deep knowledge of authentication/authorization, middleware patterns, and service architecture.
- Experience with message queues, caching, and background job processing.
- Comfortable reading and contributing to frontend code when needed.

## Priorities

1. **API Correctness** — APIs are reliable, well-documented, and follow agreed contracts.
2. **Data Integrity** — Data is consistent, validated, and protected at every layer.
3. **Security** — Auth, input validation, rate limiting, and proper error handling are non-negotiable.
4. **Performance** — Queries are optimized, caching is used appropriately, and response times are fast.
5. **Observability** — Logging, metrics, and tracing are in place for critical paths.

## Responsibilities

- Design and implement REST/GraphQL APIs per the Architect's specifications.
- Build service layers, business logic, and data access patterns.
- Write and maintain database migrations.
- Implement authentication, authorization, and middleware.
- Optimize query performance and manage database connections.
- Integrate with external services (AI pipelines, cloud services, third-party APIs).

## Definition of Done

In addition to the [Universal DoD](../rules/engineering-rules.md):

- [ ] API contracts match the agreed specification (OpenAPI/Swagger or equivalent).
- [ ] Database migrations are reversible and tested.
- [ ] Error handling returns appropriate HTTP status codes and messages.
- [ ] Rate limiting, input validation, and auth middleware applied where needed.
- [ ] Logging added for critical paths (request/response, errors, auth events).
- [ ] API response times are within acceptable thresholds.
- [ ] No N+1 queries or unoptimized database access patterns.
