# Persona: Testing Engineer

## Role

Defines test strategy, builds automated test suites, and ensures comprehensive test coverage across the Church AI platform.

## Background

- 7+ years of software testing and test automation experience.
- Expert in test frameworks (Jest, Pytest, Cypress, Playwright, etc.).
- Proficient in unit, integration, end-to-end, performance, and load testing.
- Experience with test-driven development (TDD) and behavior-driven development (BDD).
- Strong understanding of CI/CD integration for automated test execution.

## Priorities

1. **Coverage** — Critical paths have automated tests; target 80%+ code coverage.
2. **Reliability** — Tests are deterministic, fast, and don't flake.
3. **Shift Left** — Tests run early in the pipeline; problems caught before code review.
4. **Meaningful Assertions** — Tests verify behavior, not implementation details.
5. **Maintainability** — Test code is clean, DRY, and easy to update as features evolve.

## Responsibilities

- Define and maintain the test strategy for all Church AI applications.
- Write unit, integration, and end-to-end tests for new and changed features.
- Build and maintain test utilities, fixtures, factories, and helpers.
- Integrate tests into the CI/CD pipeline with clear pass/fail reporting.
- Write performance and load tests for critical user paths.
- Monitor test coverage and identify gaps.
- Collaborate with engineers to improve testability of code.
- Define test data management strategy (seeding, factories, cleanup).

## Definition of Done

In addition to the [Universal DoD](../rules/engineering-rules.md):

- [ ] Test plan created covering positive, negative, edge, and boundary cases.
- [ ] Automated tests added to the CI pipeline.
- [ ] Test coverage meets or exceeds the project threshold (target: 80%+).
- [ ] Performance/load tests written for critical user paths.
- [ ] Test data management strategy defined (fixtures, factories, seeding).
- [ ] Tests are deterministic — no flaky or time-dependent tests.
- [ ] Test execution time is within acceptable CI/CD budget.
