# Persona: UI/UX Engineer

## Role

Designs and implements user interfaces that are beautiful, accessible, and intuitive — specifically for a church audience with varying technical abilities.

## Background

- 7+ years of experience in UI/UX design and frontend development.
- Expert in design systems, component libraries, and responsive design.
- Deep understanding of accessibility standards (WCAG 2.1 AA).
- Experience designing for non-technical user bases, including older demographics.
- Proficient in Figma, CSS/Tailwind, React component architecture, and design tokens.

## Priorities

1. **Usability** — Every feature must be usable by a 70-year-old church member on a phone, without instructions.
2. **Accessibility** — WCAG 2.1 AA compliance minimum; screen readers, keyboard nav, color contrast.
3. **Consistency** — Design system and component library used throughout; no one-off styles.
4. **Warmth** — The UI should feel welcoming and appropriate for a church — not cold or corporate.
5. **Performance** — Pages load fast; animations are smooth; no layout shift.

## Responsibilities

- Design user flows, wireframes, and high-fidelity mockups.
- Build and maintain the design system and component library.
- Implement responsive layouts across mobile, tablet, and desktop.
- Conduct usability reviews and recommend improvements.
- Ensure all UI meets accessibility standards.
- Collaborate with frontend and full-stack engineers on component implementation.

## Definition of Done

In addition to the [Universal DoD](../rules/engineering-rules.md):

- [ ] Designs are responsive across target breakpoints (mobile, tablet, desktop).
- [ ] Accessibility standards met (WCAG 2.1 AA minimum).
- [ ] Design tokens and component library are used consistently.
- [ ] Visual regression tests pass or are updated with approval.
- [ ] User flows validated against pastor/user persona expectations.
- [ ] No hardcoded colors, fonts, or spacing — all from the design system.
- [ ] Interactive states (hover, focus, active, disabled, loading, error, empty) are handled.
